Spring boot 
- 스프링 프레임워크 기반의 애플리케이션을 빠르고 쉽게 개발할수 잇도록 도와주는 프레임워크 .
내장웹서버(Tomcat) 을 제공하여 독립 실행 가능한 애플리케이션을 만들 수 있다 .


내장웹서버 의 역할

웹 애플리케이션 서버 기능 제공:
별도의 웹 서버 설치 없이 애플리케이션 내부에 웹 서버 기능을 포함합니다

서블릿 컨테이너 제공:
Java 서블릿을 실행하고 관리하는 환경을 제공합니다3.

HTTP 요청 처리
클라이언트로부터의 HTTP 요청을 받아 처리하고 응답을 반환합니다

포트 바인딩
기본적으로 8080 포트에 바인딩되어 웹 요청을 수신합니다

자동 구성:
SpringBoot가 제공하는 자동 구성 기능을 통해 별도의 복잡한 설정 없이 웹 서버를 구동할 수 있습니다

이러한특징들로 부트는 개발자가 웹서버설정과 배포에 대한 복잡성을 줄이고 비즈니스 로직개발에 더 집중할 수 있게도와준다 .

독립실행이란 ? 
별도의 웹서버없이 실행가능 - 자체적으로 실행할 수 있다 .
단일 JAR파일로 배포 - 애플리케이션과 모든 의존성을 포함하여 단일 JAR 파일로 패키징되어 java-jar 명령으로 실행할 수 있다 .
환경 독립성 : 개발 테스트 운영환경에 관계없이 동일한방식으로 애플리케이션을 실행할 수 있다 .

독립실행 가능한 애플리케이션은 특히 보안 , 성능 ,사용 편의성이 중요한 상황에서 유용하게 활용될수있다 .

REST : Representational State Transfer
REST 는 웹 서비스 아키텍처 스타일로 주로 HTTP 프로토콜을 통해 자원을 주고받는 방식을정의한다 .
RESTful 시스템의 원칙
1 .리소스 기반 URI는 자원을 나타내며 각 자원은 고유한URI를 가진다 .
2. HTTP 메서드 사용 CRUD 작업에 맞춰 GET , POST PUT DELETE 등의 메서드를 사용
3. 무상태성 : 서버는 클라이언트의 상태를 저장하지 않고 각 요청이 독립적이다 .
4. 캐시가능 : 클라리언트는 서버응답을 캐시할 수 있어 성능을 향상시킬 수 있다 .

레이어드 아키텍처 : 소프트웨어 시스템을 여러계층으로 분리하여 설계하는 아키텍처 패턴이다 .
일반적으로 4-tier 레이어드 아키텍처는 다음과 같은 계층으로 구성된다 .
Presentation Layer 
Business Layer - 구현시 고려사항중 예외처리에대해 잘 작성해보자 .
비즈니스 로직처리 : 시스템의 핵심기능과 규칙을 구현
데이터 유효성 검증 : 입력된 데이터가 비즈니스 규칙에 부합하는지 확인
트랜잭션 관리 : 데이터의 일관성과 무결성을 보장하기 위해 트랜잭션을 관리 .
계층 간 중재 : 프레젠테이션 레이어와 데이터 엑세스 레이어사이의 중간역할 수행 

Persistence Layer - 데이터의 영속성을 처리하는 계층 , DB와의 상호작용 담당 , ORM 프레임워크나 데이터접근객체(DAO) 가 이 계층에 속한다 . 
Database Layer - 실제 데이터베이스 시스템을의미한다 . 데이터를 물리적으로 저장하고 관리한다 .DBMS가 여기에 해당
